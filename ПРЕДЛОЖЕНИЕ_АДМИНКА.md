# Предложение по админке для сайта КБ Парус

**Дата:** 10 мая 2026  
**Контекст:** Сайт работает на Next.js + Vercel. Контент сейчас редактируется через TypeScript-файлы — каждое изменение требует разработчика и деплоя. Маркетолог и менеджеры не могут самостоятельно править описания, цены, добавлять товары.

---

## Когда лучше всего внедрять админку

**Сейчас.** Вот почему:

- Каталог расширяется. Уже 21 товар. Через месяц будет 50+. Править через код — медленно и дорого.
- Контент-менеджер компании должен видеть, что меняет, и видеть результат сразу.
- SEO-поля (заголовки, описания, ключевые слова, OG-картинки) — должны быть в каждой карточке. Маркетолог должен иметь к ним доступ.
- Цены, акции, новые опции — мобильность важна.
- Реальные фото оборудования будут прибывать партиями — фотографу не нужно каждый раз отправлять файл программисту.

**Что НЕ надо делать сейчас:**
- Не строить мегасложную админку с ролями, аудитом, версионированием. Это нужно через год, когда станет понятно, чего реально не хватает.
- Не лезть в Битрикс/1С — это другая задача (учётная, не контентная).

## 4 варианта админки — сравнение

### Вариант 1. Sanity Studio (рекомендую)

**Что это:** Облачная headless CMS. Для редакторов — современный веб-интерфейс. Для разработчика — гибкая схема данных в коде.

| Плюсы | Минусы |
|-------|--------|
| ✅ Бесплатно до 3 редакторов и 100 000 документов | ❌ Зависимость от Sanity (US-компания, но работает в РФ) |
| ✅ Setup за 1 час | ❌ Контент в их облаке, не в нашем git |
| ✅ Live preview — редактор видит изменения сразу | |
| ✅ Великолепный UX редактора | |
| ✅ Встроенная поддержка изображений с авто-CDN и обрезкой | |
| ✅ Real-time collaboration | |
| ✅ Полный русский язык интерфейса | |

**Стоимость:** **0 ₽/мес** на нашем уровне трафика  
**Время на запуск:** 1-2 рабочих дня  
**Кому подходит:** маркетологу без техфона, фотографу

---

### Вариант 2. Decap CMS (бывший Netlify CMS)

**Что это:** Open-source git-based CMS. Редактор правит контент через веб-форму, изменения коммитятся прямо в GitHub.

| Плюсы | Минусы |
|-------|--------|
| ✅ Полностью бесплатно | ❌ UX заметно слабее, чем у Sanity |
| ✅ Контент остаётся в нашем git | ❌ Нет live preview |
| ✅ Нет внешних зависимостей | ❌ Каждое изменение = коммит = деплой (~2 мин) |
| ✅ Любой git-сотрудник может откатить изменения | ❌ Нет real-time collaboration |
| ✅ Работает без серверов | ❌ Картинки сохраняются в репо (раздувание) |

**Стоимость:** **0 ₽/мес**  
**Время на запуск:** 1 день  
**Кому подходит:** если важна полная независимость от внешних сервисов

---

### Вариант 3. Payload CMS (open-source)

**Что это:** Современная open-source CMS на Node + TypeScript. Хостинг на нашей стороне (Vercel + бесплатная Postgres от Neon/Supabase).

| Плюсы | Минусы |
|-------|--------|
| ✅ Полный контроль, всё на нашей инфраструктуре | ❌ Сложнее в настройке |
| ✅ TypeScript-first — идеально для нашего стека | ❌ Нужна БД (бесплатно через Neon) |
| ✅ Встроенный Admin UI | ❌ ~30 часов работы программиста на запуск |
| ✅ Можно расширять как угодно | ❌ Поддержка ложится на нашего разработчика |
| ✅ Поддержка ролей, ревизий, локализации | |

**Стоимость:** **0 ₽** (open-source) + ~$0-10/мес за хостинг БД  
**Время на запуск:** 3-5 дней  
**Кому подходит:** если планируется большая команда, сложные роли, и есть постоянный разработчик

---

### Вариант 4. Strapi (open-source)

**Что это:** Самая популярная open-source headless CMS. Похожа на Payload, но появилась раньше.

| Плюсы | Минусы |
|-------|--------|
| ✅ Огромное сообщество и плагины | ❌ Тяжелее Payload |
| ✅ Готовые плагины для SEO, медиа, форм | ❌ Большой docker-контейнер для хостинга |
| ✅ Хорошая документация | ❌ Платное хостинговое решение от Strapi (~$10-30/мес) |
| | ❌ Не TypeScript-first |

**Стоимость:** **0 ₽** (self-hosted) или $10-30/мес (Strapi Cloud)  
**Время на запуск:** 3-5 дней

## ОБНОВЛЕНИЕ ОТ 10.05.2026 — финальная рекомендация: Payload CMS

После более глубокого анализа (учёт санкционного риска, Excel-калькулятор с 32 листами и сложными таблицами коэффициентов, требование российской компании владеть данными) — **итоговая рекомендация изменилась с Sanity на Payload CMS**. См. план `~/.claude/plans/peaceful-beaming-scone.md` и расширенное описание ниже.

Краткое сравнение в новой ситуации:

- **Sanity** — отпадает: US-SaaS, оплата в USD из РФ нестабильна, контент на их серверах = риск потери доступа.
- **Decap** — отпадает: маркетолог не сможет редактировать 30+ строк таблиц коэффициентов через GitHub PR.
- **Strapi/Directus** — рабочие, но требуют отдельный сервер и не TypeScript-first.
- **Payload CMS 3** — победитель. Живёт ВНУТРИ нашего Next.js-приложения, бесплатный, TS-first, поддерживается на Vercel.

Стоимость первого года: **0 ₽** (Vercel hosting + Neon Postgres free + Vercel Blob free).

## Что НЕ будет в CMS (остаётся в коде)
- Формулы калькулятора в `lib/calculator/pricing.ts` (физика и инженерия — не маркетинговый текст)
- Discriminated `pricing.kind` профиля (меняет ветку формулы, маркетолог видит read-only)
- API заявок и интеграция с Telegram/Bitrix24
- Защита форм (honeypot, rate-limit)

## Перенос на купленный домен — работает БЕЗ ПРОБЛЕМ

Postgres БД и Blob хранилище **независимы от домена**. Когда вы купите свой домен:
1. В Vercel Dashboard → Settings → Domains → Add → ввести домен
2. Прописать DNS-записи у регистратора
3. Через 10-30 минут сайт открывается по новому адресу
4. Postgres и Blob продолжают работать без изменений

Если когда-нибудь захотите уйти с Vercel — экспорт `pg_dump` (один файл) + копия Blob = переезд за полдня на любой VPS.

## Старая рекомендация (НЕ ИСПОЛНЯТЬ): Sanity

**Почему именно Sanity:**

1. **Самый быстрый запуск.** За 1-2 дня админка будет работать. Маркетолог сможет редактировать в этот же день.
2. **Бесплатно.** До 3 редакторов хватит на ближайший год точно.
3. **Лучший UX для не-технических редакторов.** Маркетолог поймёт интерфейс за 30 минут.
4. **Изображения.** Загрузка, обрезка, авто-CDN, оптимизация — всё из коробки. Это критично для нас (стоковые фото нужно заменять).
5. **Live preview.** Редактор изменил описание — сразу видит результат на сайте без деплоя.
6. **Real-time collaboration.** Несколько человек могут работать параллельно.

**Когда переходить с Sanity:**
- Если приобретёт более 3 редакторов → апгрейд $99/мес или переход на Payload
- Если нужны сложные роли → переход на Payload
- Если нужна полная независимость → Decap CMS или Payload

В обозримом будущем (1-2 года) такой потребности у КБ Парус не будет.

## План внедрения Sanity (поэтапно)

### Этап 1 (1 день, сегодня) — Подготовка кодовой базы

**Что делаем:**
1. Расширяем TypeScript-типы данных каталога: добавляем поля `seoTitle`, `seoDescription`, `ogImage`, `keywords`, `noIndex`, `featured`, `priceFrom`, `priceTo`, `priceLabel`
2. Все эти поля ОПЦИОНАЛЬНЫЕ с разумными дефолтами
3. Сайт продолжает работать на TypeScript-данных

**Результат:** структура данных готова к миграции в CMS, но до миграции редактор пишет в коде.

### Этап 2 (1-2 дня) — Создание Sanity-проекта

**Что делаем:**
1. Регистрация Sanity-проекта (бесплатный план)
2. Создание схем данных в Sanity Studio:
   - Категория (17 шт.)
   - Подкатегория
   - Товар с галереей и спецификациями
   - Глобальные настройки (контакты, SEO-дефолты)
   - FAQ
   - Кейсы
3. Импорт текущего контента (одной командой через скрипт)
4. Настройка ролей: «Главный редактор» (всё), «Контент-редактор» (без удаления), «Фотограф» (только медиа)

**Результат:** Sanity Studio работает на отдельном поддомене (например, `studio.kbparus-metal-storage.vercel.app`). Редактор может зайти и видеть контент.

### Этап 3 (1 день) — Подключение сайта к CMS

**Что делаем:**
1. На страницах сайта меняем источник данных с TypeScript-файлов на Sanity API
2. Кэширование на стороне Next.js: ISR (Incremental Static Regeneration) — сайт быстрый, но обновляется через 1 минуту после изменения в админке
3. Live preview для редакторов: они видят будущую версию сайта до публикации

**Результат:** маркетолог редактирует в админке → через минуту изменения на сайте.

### Этап 4 (полдня) — Обучение и документация

**Что делаем:**
1. Скринкаст «Как работать с Sanity Studio» (5-7 минут)
2. Документ «Как добавить новый товар» (1 страница)
3. Документ «Как заменить фото» (полстраницы)
4. Передача редактору (маркетологу/менеджеру)

**Результат:** компания самостоятельна в управлении контентом.

## Что выходит «из коробки» с Sanity

После внедрения редактор сможет:

- ✅ Добавлять/удалять/редактировать категории каталога
- ✅ Создавать новые товары с галереей фото
- ✅ Менять характеристики, описания, цены
- ✅ Загружать фотографии (с авто-обрезкой и оптимизацией)
- ✅ Редактировать тексты главной страницы (hero, преимущества, кейсы, отзывы, FAQ)
- ✅ Менять SEO-метаданные каждой страницы (заголовок, описание, ключевые слова, OG-картинка)
- ✅ Управлять баннерами на kbparus.ru и линииокраски.рф
- ✅ Видеть изменения в режиме предпросмотра до публикации
- ✅ Откатываться к предыдущим версиям контента

**Что НЕ автоматизируется в Этапе 4** (можно добавить позже):
- Управление контактами и менеджерами форм заявок (это в коде)
- Редактирование структуры калькулятора (формулы) — это бизнес-логика
- Аналитика заявок (это в Telegram + Bitrix)
