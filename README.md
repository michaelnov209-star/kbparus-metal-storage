# КБ Парус: сайт «Системы хранения металла»

Next.js/React MVP промышленного B2B-сайта для подбора систем хранения металла. Проект задуман как основа боевого продукта: контент вынесен в данные, расчёты отделены от UI, заявки готовы к передаче в Bitrix24, а структура оборудования подготовлена под будущую CMS и 1С.

Production URL: https://kbparus-metal-storage.vercel.app

## Что сейчас есть на сайте

- Полностью пересобранный premium hero: тёмная промышленная сцена, крупная типографика, продуктовый visual-layer, логотип КБ Парус, CTA на расчёт и связь с инженером.
- Блок «О компании» с коротким клиентским текстом о КБ Парус: с 2009 года, проектирование, производство, автоматизация и инженерные решения.
- Сокращённый маршрут страницы: проблема склада → что храним → типы систем → до/после → инженерный конфигуратор → проект → преимущества → контакты.
- Блок «что можно хранить»: лист, трубы, профиль, сортовой прокат, смешанные партии, тяжёлые пачки.
- Интерактивный showcase типов систем хранения.
- Интерактивный showcase решений и premium-карточки систем хранения с крупными временными фото.
- Блок «до / после»: хаос и металл на полу против организованной системы хранения.
- Блок «как это работает»: хранение → доступ → перемещение → загрузка.
- Многошаговый инженерный конфигуратор с карточками выбора, прогрессом, live summary, визуальным превью конфигурации, анимацией стартовой суммы «от» и CTA «Получить инженерный расчёт». Сложные инженерные поля спрятаны в раскрываемый блок.
- Блок «что получает клиент после расчёта».
- Блок проектирования: данные, подбор, нагрузка, конструкция, производство, доставка и монтаж.
- Блок преимуществ, сценариев применения, данных для инженера и расширенный FAQ.

## Структура проекта

- `app/` — Next.js App Router, главная страница, layout, favicon/icon и API route.
- `app/api/leads/route.ts` — серверный endpoint заявки.
- `components/` — UI-компоненты: бренд, hero visual, showcase решений, калькулятор.
- `data/storageSystems/` — клиентский контент, оборудование, категории, коэффициенты, опции, FAQ, контакты, ссылки и визуальные ассеты.
- `lib/calculator/` — расчётная логика, типы, рекомендация системы, pricing, форматирование и нормализация входных данных.
- `public/brand/` — брендовые ассеты.
- `public/assets/images/` — папка для будущих локальных фото систем хранения и склада.
- `tests/` — unit-тесты калькулятора.

## Где что лежит

- Логотип: `public/brand/logo-g.png`
- Favicon: `app/icon.svg`
- Визуальные ассеты сайта: временные remote JPEG из Pexels + локальная папка `public/assets/images/` под будущие фото КБ Парус.
- Карта подключённых изображений и источников: `data/storageSystems/visualAssets.ts`
- Папка под будущие реальные изображения: `public/assets/images/`
- Данные оборудования: `data/storageSystems/equipment.ts`
- Категории решений: `data/storageSystems/productCategories.ts`
- Контакты, FAQ, тексты блоков, сценарии и данные для инженера: `data/storageSystems/content.ts`
- Опции калькулятора: `data/storageSystems/calculatorOptions.ts`
- Коэффициенты расчёта: `data/storageSystems/priceFactors.ts`
- Логика калькулятора: `lib/calculator/`
- API заявки: `app/api/leads/route.ts`
- Bitrix24 webhook: переменная окружения `BITRIX24_WEBHOOK_URL`
- Будущая CMS: заменить файлы `data/storageSystems/*` адаптером Payload CMS, Directus, Strapi или другой headless CMS.
- Будущая 1С: поля для интеграции уже есть в `equipment.ts` (`product_id`, `sku`, `external_1c_id`, цены, НДС, вес, габариты, тип изделия).

## Архитектурные решения

- Контент не должен хардкодиться внутри UI-компонентов. Основной контент хранится в `data/storageSystems`.
- Расчёты калькулятора отделены от UI и лежат в `lib/calculator`.
- Клиентские тексты отделены от технической логики и внутренних идентификаторов.
- В интерфейсе не показываются внутренние термины вроде `SKU`, `product_id`, `external_1c_id`.
- Калькулятор не обещает точную цену: в интерфейсе используется стартовая сумма «от», «ориентир для первичного подбора», «финальная стоимость уточняется после инженерной проверки».
- API заявки передаёт структурированные параметры, чтобы менеджер мог быстрее готовить предложение.
- Проект готовится под CMS, Bitrix24 и будущую 1С, но не зависит от них в MVP.
