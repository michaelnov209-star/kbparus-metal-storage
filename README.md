# КБ Парус: сайт «Системы хранения металла»

Next.js/React сайт для направления систем хранения металла. Проект собирается как B2B-сайт-инструмент: клиент видит каталог из Excel, выбирает параметры в калькуляторе, получает стартовую сумму «от» и отправляет заявку инженеру.

Production URL: https://kbparus-metal-storage.vercel.app

## Что сейчас есть на сайте

- Главная страница в структуре сайта по линиям окраски: hero, каталог, калькулятор стоимости, CTA-баннеры, кейсы, география, отзывы, о компании, преимущества, 3 шага, партнёры, FAQ, форма связи, контакты, карта и футер.
- Светлая промышленная тема: белая база, графитовые акценты, оранжевый `#fc5413`, крупная типографика, карточки, hover-состояния и scroll-анимации.
- Header с логотипом, выпадающей кнопкой «Каталог», навигацией и двумя телефонами:
  - +7 (499) 403-39-62
  - +7 (495) 741-87-10
- Hero с крупным заголовком, pseudo-video/фото-блоком и метриками:
  - 1,5 млн м² выполненных работ
  - 3000+ довольных покупателей
  - 700+ городов обслуживания
  - 15+ лет производим оборудование
- Каталог на главной строго соответствует видимым ячейкам листа `Главная` в `Калькулятор-New.xls`.
- Калькулятор стоимости работает по Excel-профилям: фиксированные варианты высоты, ширины, длины, нагрузки, количества полок, башен и опций.
- API `/api/leads` принимает заявки, параметры калькулятора и в mock-режиме отвечает без `BITRIX24_WEBHOOK_URL`.
- Используются реальные отображаемые remote-фото через `<img>`, вынесенные в `data/storageSystems/visualAssets.ts`.

## Структура проекта

- `app/` — Next.js App Router, главная страница, layout, icon и API route.
- `app/api/leads/route.ts` — endpoint заявки.
- `components/` — UI-компоненты: логотип, калькулятор, форма заявки, page styles.
- `data/storageSystems/` — контент, контакты, визуальные ассеты, Excel-каталог и Excel-профили калькулятора.
- `lib/calculator/` — расчётная логика, типы, нормализация, форматирование.
- `public/brand/` — логотип.
- `public/assets/images/` — место для будущих локальных фото/визуалов.
- `tests/` — unit-тесты калькулятора.
- `docs/` — handoff и проектная документация для следующего разработчика/агента.

## Где что лежит

- Логотип: `public/brand/logo-g.png`
- Favicon: `app/icon.svg`

- Логика расчёта: `lib/calculator/pricing.ts`
- Типы калькулятора: `lib/calculator/types.ts`
- Контакты, FAQ, кейсы, преимущества: `data/storageSystems/content.ts`
- Форма связи: `components/LeadForm.tsx`
- Bitrix24 webhook: env `BITRIX24_WEBHOOK_URL`
- Будущая CMS: заменить `data/storageSystems/*` адаптером Payload CMS / Directus / Strapi.
- Будущая 1С: данные сохраняются структурно, без расчёта только текстом.

## Excel-логика

Источник: `C:/Users/micha/Downloads/Калькулятор-New.xls`

- `Админка` — цены, коэффициенты, опции и значения, которые в будущем должны управляться через CMS/admin UI.
- `Главная` — публичный каталог на главной странице. Сайт не добавляет разделы сверх этого листа.
- Формульные листы — профили калькулятора:
  - `Авт.скл. лист. металл`
  - `Авт.скл.сорт мет.`
  - `Стеллаж с выкатными кассетами`
  - `Касетный под погрузчик`
  - `Двухсторонний стеллаж с вык. к.`
  - `Гибридный стеллаж`

Интерфейс не показывает старые следы из Excel про печь полимеризации или линии порошковой окраски.

## Архитектурные решения

- Контент не хардкодится в компонентах, а хранится в `data/storageSystems`.
- Расчёты не находятся в UI, а лежат в `lib/calculator`.
- Клиентские тексты отделены от технической логики.
- В интерфейсе не показываются `SKU`, `product_id`, `external_1c_id`.
- MVP готовится под CMS, Bitrix24 и будущую интеграцию с 1С.
