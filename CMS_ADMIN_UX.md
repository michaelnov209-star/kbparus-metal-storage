# CMS admin UX

## Цель

Payload admin должен работать как операционный центр управления сайтом КБ Парус, а не как набор технических таблиц.

## Логика навигации

- Главная страница: модульное управление лендингом, hero, преимуществами, кейсами, отзывами, FAQ, партнёрами и баннерами.
- Каталог: категории, подкатегории и товары. URL-поля считаются чувствительными для SEO и внешних ссылок.
- Калькулятор: профили расчётов, цены, коэффициенты и опции, которые влияют на стоимость и Telegram-заявку.
- Компания: контакты, реквизиты, мессенджеры и сквозные данные сайта.
- Медиа: фото, видео, баннеры, логотипы и документы.
- Заявки: операционная карта форм, Telegram и подготовка Bitrix24.
- Пользователи: сотрудники админки, роли и доступы.

## Что изменено для менеджеров

- Добавлен dashboard "Центр управления сайтом и каталогом" с быстрыми переходами.
- Коллекции и globals сгруппированы по бизнес-разделам.
- Labels и descriptions переведены на деловой русский язык.
- Главная страница описана как редактор лендинга по порядку блоков.
- Медиа-библиотека получила тип материала, область использования и заметки редактора.
- Пользователи получили раздельные поля: имя, фамилия, отображаемое имя, должность.
- Добавлен раздел "Заявки: формы и интеграции" без секретов и без изменения env.

## Performance findings

До изменений production admin document requests занимали около 6.5-7 с после простоя:

- `/admin`: около 6.6 с.
- `/admin/collections/products`: около 6.5 с.
- `/api/health`: около 7.0 с.
- `/`: около 1.0 с.

Такой профиль указывает не на CSS или bundle как главную причину, а на serverless cold start и подключение к Neon/Payload runtime. Лёгкие list views всё равно добавлены: меньше relation columns, меньше default pagination, меньше дорогих превью в списках.

## Текущие безопасные оптимизации

- Убраны тяжёлые relationship columns из списков товаров и подкатегорий.
- Ограничены default list limits.
- Добавлены `listSearchableFields` для предсказуемого поиска.
- Dashboard сделан статическим server component без клиентского JS.
- Hero-видео заменено на оптимизированный MP4 без изменения URL.
