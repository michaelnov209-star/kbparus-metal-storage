# Project Context

Проект: сайт направления КБ Парус «Системы хранения металла».

Production URL: https://kbparus-metal-storage.vercel.app

GitHub: https://github.com/michaelnov209-star/kbparus-metal-storage

Основная ветка: `main`

## Цель проекта

Сайт должен работать как B2B-инструмент продаж и первичного подбора оборудования:

- показать направление «Системы хранения металла» как промышленный продукт КБ Парус;
- объяснить, какие системы подходят под лист, трубу, профиль, балки и смешанный металл;
- дать клиенту понятный калькулятор первичного подбора;
- собрать заявку и передать параметры в CRM;
- сохранить архитектуру под будущую CMS, Bitrix24 и 1С.

## Важные продуктовые ограничения

- Калькулятор не должен обещать точную цену.
- В интерфейсе использовать формулировки: «от ...», «стартовая сумма», «ориентир для первичного подбора», «финальная стоимость уточняется после инженерной проверки».
- В клиентском интерфейсе не показывать внутренние термины вроде `sku`, `product_id`, `external_1c_id`, НДС и технические поля интеграции.
- Конкурентные ссылки и референсы можно хранить в данных, но публичный блок «конкуренты» на сайте не нужен.
