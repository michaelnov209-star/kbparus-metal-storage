# Internal lead inbox

Дата: 2026-05-20

## Что это

`leads` — внутренняя коллекция Payload CMS для заявок с сайта. Она снижает зависимость от внешней CRM: даже если Bitrix24 временно отключен или Telegram недоступен, заявка может быть сохранена в админке.

Адрес в админке:

```text
/admin/collections/leads
```

## Что сохраняется

- тип заявки: контактная форма или конфигуратор;
- имя, телефон, email, город;
- комментарий клиента;
- источник: страница, товар, URL;
- UTM-метки;
- параметры калькулятора;
- рекомендованная система и ориентировочная цена;
- статус доставки на email;
- статусы доставки в Telegram / Bitrix24;
- статус обработки менеджером.

## Статусы

| Статус | Когда использовать |
|---|---|
| `new` | Новая заявка, еще не обработана |
| `in_progress` | Менеджер взял в работу |
| `contacted` | С клиентом связались |
| `closed` | Заявка закрыта |
| `spam` | Спам или мусорная заявка |

## Поведение API

`/api/leads` теперь пишет заявку в CMS после попыток внешней доставки. Telegram и Bitrix24 остаются независимыми каналами.

Если Bitrix24 не работает, это не блокирует Telegram и внутренний журнал. Если Telegram недоступен, заявка все равно может сохраниться в CMS.

## Проверка

1. Отправить тестовую заявку через форму.
2. Открыть `/admin/collections/leads`.
3. Проверить новую запись: телефон, источник, UTM, конфигурация калькулятора, delivery flags.
4. Поменять статус на `in_progress` или `contacted`.

## Ограничения

- Коллекция не заменяет полноценную CRM: нет воронки, задач, ответственных и SLA.
- Это надежный внутренний журнал, пока Bitrix24 отложен.
- Публичное создание через Payload REST закрыто; заявки создаются серверным `/api/leads`.
