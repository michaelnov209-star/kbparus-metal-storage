# 🤖 Как настроить Telegram-бота для заявок с сайта

**Время на настройку:** 10–15 минут  
**Что вы получите:** все заявки с сайта будут приходить в закрытую Telegram-группу, где сидят только нужные люди (менеджеры, инженеры). Каждый увидит заявку в течение 1–2 секунд после отправки.

---

## Зачем это нужно

Сейчас клиент заполняет форму на сайте → видит «Заявка отправлена» → но никто из команды не знает, что появилась новая заявка. Заявка попадает в Bitrix24 (если настроен) или просто теряется.

После настройки Telegram-бота:
- ✅ В закрытую группу падает уведомление о каждой заявке
- ✅ В уведомлении: имя, телефон, город, тип оборудования, цена, комментарий
- ✅ Телефон кликабельный — нажал и сразу звонишь
- ✅ Заявка дублируется в Bitrix24 (если настроен) — то есть у вас будет два канала
- ✅ Бесплатно, не требует серверов

---

## Шаг 1: Создаём бота через @BotFather

1. Откройте Telegram, найдите бота **@BotFather** (официальный бот Telegram)
2. Нажмите **Start** (или отправьте команду `/start`)
3. Отправьте команду `/newbot`
4. BotFather спросит имя бота (как будет отображаться в чатах). Например:
   ```
   КБ Парус Заявки
   ```
5. BotFather спросит username бота — должно заканчиваться на `bot` и быть уникальным. Например:
   ```
   kbparus_leads_bot
   ```
6. BotFather пришлёт сообщение с **токеном** — это длинная строка вида:
   ```
   1234567890:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```

⚠️ **СОХРАНИТЕ ЭТОТ ТОКЕН** — он даёт полный контроль над ботом. Не показывайте никому. Если случайно показали — попросите у BotFather новый через `/revoke`.

---

## Шаг 2: Создаём закрытую группу для заявок

1. В Telegram: **Меню → Создать группу**
2. Название: например, **«КБ Парус — Заявки с сайта»**
3. Добавьте в группу нужных людей (менеджеры, инженеры — всех, кто должен видеть заявки)
4. **Также добавьте вашего бота** (его username, например `@kbparus_leads_bot`)

После создания группы:
1. Перейдите в группу → **«Имя группы»** в шапке → **«Управлять»** → **«Администраторы»**
2. Добавьте бота администратором (минимум: «Удаление сообщений», остальное по желанию)
3. Это нужно, чтобы бот мог писать в группу

---

## Шаг 3: Узнаём chat_id группы

Это самая «техническая» часть. Бот не знает, в какой группе он живёт — нужно ему сказать.

### Способ 1: через бота @getmyid_bot (самый простой)

1. В Telegram: **найдите @getmyid_bot** (или похожих: @userinfobot, @username_to_id_bot)
2. Добавьте его в вашу новую группу как обычного участника
3. После того, как он зайдёт, он автоматически напишет ID чата — что-то вроде:
   ```
   Group ID: -1001234567890
   ```
4. **Скопируйте это число** (вместе с минусом!) — это ваш `TELEGRAM_CHAT_ID`
5. Теперь можно убрать @getmyid_bot из группы — он больше не нужен

### Способ 2: через браузер (если первый способ не работает)

1. В вашей новой группе отправьте любое сообщение (например, «тест»)
2. Откройте в браузере (замените YOUR_TOKEN на токен из шага 1):
   ```
   https://api.telegram.org/botYOUR_TOKEN/getUpdates
   ```
3. В ответе найдите блок с вашим сообщением и поле `"chat":{"id":-1001234567890,...`
4. Значение `id` — это и есть ваш `TELEGRAM_CHAT_ID`

⚠️ Для **групп** chat_id всегда **отрицательный** (начинается с минуса), для личных чатов — положительный.

## Шаг 4: Подключаем бота к сайту через Vercel

Сайт КБ Парус хостится на Vercel. Нужно добавить две переменные окружения:

1. Откройте https://vercel.com/dashboard
2. Войдите в свой аккаунт
3. Выберите проект **kbparus-metal-storage**
4. Перейдите: **Settings** (вкладка наверху) → **Environment Variables** (в боковом меню)
5. Добавьте две переменные:

   **Переменная 1:**
   - **Name:** `TELEGRAM_BOT_TOKEN`
   - **Value:** ваш токен из шага 1 (например, `1234567890:AAHxxxx...`)
   - **Environments:** отметьте все три — Production, Preview, Development
   - Нажмите **Save**

   **Переменная 2:**
   - **Name:** `TELEGRAM_CHAT_ID`
   - **Value:** ID группы из шага 3 (например, `-1001234567890`, **со знаком минус!**)
   - **Environments:** все три
   - Нажмите **Save**

6. Перейдите в **Deployments**, найдите последний deployment, нажмите **«⋯» → Redeploy** (или подождите 1-2 минуты — Vercel сам передеплоит)

## Шаг 5: Проверяем

1. Откройте сайт https://kbparus-metal-storage.vercel.app
2. Прокрутите до раздела «Контакты» или «Получить консультацию»
3. Заполните форму тестовыми данными:
   - Имя: «Тест»
   - Телефон: ваш номер (можно настоящий — это удобно для проверки)
   - Email: тестовый
   - Город: «Москва»
   - Комментарий: «Это тестовая заявка»
4. Нажмите **«Отправить заявку»**
5. **В вашей группе должно прийти сообщение от бота в течение 1-2 секунд:**

```
🔔 Новая заявка с сайта КБ Парус

👤 Имя: Тест
📞 Телефон: +7 999 000 00 00
✉️ Email: test@test.ru
📍 Город: Москва

🏗 Оборудование: Автоматизированный склад листового металла
💰 Ориентировочно: от 1 500 000 ₽

💬 Комментарий:
Это тестовая заявка

🕒 10.05.2026, 14:23 МСК
```

Если сообщение пришло — **готово, всё работает**. Если нет — см. раздел «Что-то не работает» внизу.

## Что-то не работает

### Заявка отправлена, но в группу ничего не пришло

**Причина 1: токен или chat_id указаны неправильно**

Откройте в браузере (замените YOUR_TOKEN):
```
https://api.telegram.org/botYOUR_TOKEN/getMe
```
- Если показывает данные бота (имя, username) — токен правильный
- Если показывает ошибку 401 — токен неверный

**Причина 2: бот не добавлен в группу или не админ**

В Telegram: **группа → шапка → Управлять → Участники** — проверьте, что бот там и он администратор.

**Причина 3: Vercel не передеплоил после изменения переменных**

В Vercel Dashboard → Deployments → нажмите Redeploy на последнем deployment.

**Причина 4: chat_id указан без минуса**

Для групп chat_id ВСЕГДА начинается с минуса. `-1001234567890`, не `1001234567890`.

### В группу пришла заявка, но без ссылок и форматирования

Проверьте в Vercel, что в значении `TELEGRAM_BOT_TOKEN` нет лишних пробелов в начале или конце.

### Хочу несколько групп (отдельно для разных направлений)

Сейчас бот пишет в одну группу. Чтобы было несколько:
- Либо создайте отдельных ботов для разных направлений
- Либо добавьте логику разбора заявок (например, по типу оборудования) — это требует доработки кода, обратитесь к разработчику

## Дополнительные возможности (на будущее)

После того как базовая настройка работает, можно расширить:

### 1. Кнопки «Принять заявку» / «Отказать» в Telegram

Менеджер прямо в группе нажимает кнопку — статус меняется в CRM. Требует доработки кода (около 4 часов работы).

### 2. Автоматическое создание задачи в Bitrix24

Уже частично работает: если установить ОБА токена (`TELEGRAM_BOT_TOKEN` и `BITRIX24_WEBHOOK_URL`) — заявки идут в оба места одновременно.

### 3. Уведомление о статусе обработки

Когда заявка обработана в CRM — бот пишет в группу «✅ Обработано». Требует доработки кода.

### 4. Ежедневная сводка

В 9 утра бот присылает: «Вчера было 5 заявок, обработано 3, в работе 2». Требует доработки кода.
