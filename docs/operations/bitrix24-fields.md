# Bitrix24: поля для заявок с сайта

Дата: 2026-05-19

## Что уже отправляется без custom fields

Интеграция работает через `BITRIX24_WEBHOOK_URL`. Если webhook не задан, Bitrix24 пропускается, Telegram и mock-режим продолжают работать.

Стандартные поля Bitrix24:

| Поле | Что передается |
|---|---|
| `TITLE` | Тип заявки и товар/страница |
| `NAME` | Имя клиента |
| `PHONE` | Телефон |
| `EMAIL` | Email, если указан |
| `COMMENTS` | Комментарий, страница, город, UTM, конфигурация калькулятора, ориентировочная цена |
| `SOURCE_ID` | `WEB` |
| `SOURCE_DESCRIPTION` | URL или название источника |

Это минимально безопасный режим: даже без настройки дополнительных полей менеджер видит полный контекст в комментарии.

## Рекомендуемые custom fields

Создать или проверить поля в CRM-карточке сделки/лида и прописать их коды в Vercel env.

| Env | Тип поля в Bitrix24 | Содержание |
|---|---|---|
| `BITRIX24_FIELD_CITY` | Строка | Город/регион поставки |
| `BITRIX24_FIELD_FORM_TYPE` | Список или строка | `contact` / `configurator` |
| `BITRIX24_FIELD_SOURCE_PAGE` | Ссылка или строка | Страница, откуда пришла заявка |
| `BITRIX24_FIELD_SOURCE_TITLE` | Строка | Название страницы, товара или категории |
| `BITRIX24_FIELD_UTM` | Многострочный текст | UTM-метки |
| `BITRIX24_FIELD_CALCULATOR_CONFIG` | Многострочный текст | Конфигурация калькулятора |
| `BITRIX24_FIELD_SELECTED_PRODUCT` | Строка | Выбранный товар/рекомендованная система |
| `BITRIX24_FIELD_PRELIMINARY_PRICE_FROM` | Число | Ориентировочная цена "от" |

## Настройка webhook

1. Bitrix24 -> Разработчикам / Webhooks -> Входящий webhook.
2. Дать права на создание CRM-сущности, которую использует URL webhook.
3. В `BITRIX24_WEBHOOK_URL` указать полный метод, например:

```text
https://YOUR.bitrix24.ru/rest/USER_ID/TOKEN/crm.deal.add.json
```

Если используется `crm.lead.add.json`, стандартный payload также совместим, но поля должны быть созданы именно для лидов.

Если в env указан только базовый webhook вида `https://YOUR.bitrix24.ru/rest/USER_ID/TOKEN/`, приложение автоматически добавит `crm.deal.add.json`.

## Проверка

1. В Vercel задать `BITRIX24_WEBHOOK_URL` и нужные `BITRIX24_FIELD_*`.
2. Отправить тестовую заявку с обычной формы.
3. Отправить тестовую заявку из калькулятора.
4. Проверить, что Telegram не изменился и продолжает получать заявку.
5. В Bitrix24 проверить `COMMENTS` даже при пустых custom fields.

## Риски

- Неверный код custom field не ломает заявку, но поле не заполнится.
- Если webhook задан, но Bitrix24 недоступен, заявка не блокирует Telegram; без Telegram fallback пользователь получит ошибку доставки.
- Юридически значимые формулировки политики нужно подтвердить с юристом перед paid traffic.
