# Handoff для Claude Code: kbparus-metal-storage

## Контекст проекта

Это Next.js/React сайт направления КБ Парус «Системы хранения металла». Задача сайта — не просто витрина, а инструмент первичного подбора: клиент выбирает тип системы, ходовые размеры и опции, получает стартовую сумму `от ... ₽` и отправляет заявку инженеру.

Production: https://kbparus-metal-storage.vercel.app

GitHub: https://github.com/michaelnov209-star/kbparus-metal-storage

## Главное правило по Excel

Источник актуальной логики: `C:/Users/micha/Downloads/Калькулятор-New.xls`.

- Лист `Главная` = публичный каталог на главной странице. Не расширять каталог вручную.
- Лист `Админка` = имитация будущей админки: цены, коэффициенты, опции.
- Формульные листы = отдельные профили калькулятора.
- `1` и `0` в Excel — это имитация выбора пользователя. В коде используется нормальное состояние React.
- Старые следы про печь полимеризации, порошковую окраску и линии окраски не выводить на сайт.
- Явные опечатки можно исправлять в клиентском тексте.

- `data/storageSystems/excelCatalog.ts` — пункты каталога с листа `Главная`.
- `data/storageSystems/excelCalculator.ts` — профили калькулятора и данные из `Админка`.
- `lib/calculator/pricing.ts` — расчётная логика по формулам Excel.
- `tests/calculator.test.ts` — сверка 6 сценариев с формульными листами.

## Формульные профили калькулятора

Сейчас внедрены:

- `auto-sheet-metal` → `Авт.скл. лист. металл`
- `auto-sort-metal` → `Авт.скл.сорт мет.`
- `rollout-cassette-rack` → `Стеллаж с выкатными кассетами`
- `forklift-cassette-rack` → `Касетный под погрузчик`
- `two-side-rollout-rack` → `Двухсторонний стеллаж с вык. к.`
- `hybrid-rollout-rack` → `Гибридный стеллаж`

Калькулятор должен показывать только фиксированные значения, которые заложены в профиле: высота, ширина, длина, нагрузка, количество полок, башен и опции.

## API и интеграции

Endpoint: `app/api/leads/route.ts`

Принимает:

- contact;
- city;
- comment;
- calculatorInput;
- UTM;
- данные расчёта пересчитываются сервером.

Без `BITRIX24_WEBHOOK_URL` endpoint работает в mock-режиме.

Будущие интеграции:

- Bitrix24 через env `BITRIX24_WEBHOOK_URL`;
- CMS через замену `data/storageSystems/*` на адаптер;
- 1С через структурированные поля оборудования и расчёта.

## Проверки перед деплоем

```bash
npm run lint
npm run test
npm run build
```

После деплоя проверить:

- главная открывается;
- каталог совпадает с Excel;
- оба телефона видны в шапке;
- калькулятор переключает профили и считает `от ... ₽`;
- `/api/leads` отвечает.

## Важные замечания

- В Excel есть спорная цена опции «Весы на распалетчик» = `3 ₽`; в коде она сохранена как в книге и отмечена в README как TODO для согласования.
- В рабочей папке могут быть посторонние untracked-файлы (`app/taste`, логи, `index.html`). Не коммитить их без отдельного указания.
- Не использовать публичный блок «конкуренты» на сайте.
- Не показывать пользователю технические поля вроде `SKU`, `product_id`, `external_1c_id`.
